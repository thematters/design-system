# 雙軌工作流（Dual-Track Workflow）

前設計師離職後，為了讓「小改」能快、「結構改動」不失控，Matters 把所有設計類任務分成兩條軌道。這份文件是 PM / 運營 / 開發判斷某個任務該走哪條線的依據。

---

## 模板軌（Template Track）— 運營 / 營運可自助

### 適用情境（Use cases）

- 宣傳圖（活動圖、社群圖、Threads / IG 配圖）
- Landing page 小改：文案替換、主視覺圖替換、從既有色組裡換一組
- Email / Newsletter header 換圖換文案
- 重複性高、結構穩定的產出

### 流程

```
從 templates/ 挑模板
      │
      ▼
填變數（文案 / 圖片 / 色組合）
      │
      ▼
產出（本地預覽 / scripts/og.ts）
      │
      ▼
內部 review（PM 或 運營負責人，1 人）
      │
      ▼
發布
```

### 限制（紅線）

- 不可更動模板**結構**（排版、區塊順序、grid）
- 不可引入模板沒有的**字體家族**
- 不可使用未在 tokens 中的**主色**
- 跨出這條線 → 轉結構軌

### Review

單人複核即可。PM 或運營負責人視情境而定。

---

## 結構軌（Structural Track）— 走完整流程

### 適用情境（Use cases）

- 前端改版（作品頁、個人頁、動態、編輯器等 product pages）
- 新 Landing page（從零設計一個新活動頁）
- 視覺系統升級（新增 tokens、改主色盤、改字體系統）
- Archive 改版 / 大型資訊架構調整

### 流程

繼承前設計師時代的 12 步精神，但把角色替換：

```
1. 盤點需求（取代「大家討論」）
      │
      ▼
2. PM 認單、排序（取代「Alex 認單」）
      │
      ▼
3. 設計（Claude / Codex 輔助，PM 審稿）← 取代「設計師執行」
      │
      ▼
4. 產品 / 開發檢視（衍生問題 / 技術可行性）
      │
      ▼
5. 數次調整設計稿
      │
      ▼
6. 確認設計
      │
      ▼
7. 產 handoff spec（docs/handoff-spec-template.md）
      │
      ▼
8. 開發
      │
      ▼
9. 產品測試
      │
      ▼
10. 運營測試（視情境）
      │
      ▼
11. 依反饋 List 修 bug / 細節
      │
      ▼
12. Release
```

### Review

多方：PM、開發、運營都需過目。結構 / 流程設計由 PM 定案，技術可行性由開發守門，運營面影響由運營守門。

---

## 如何判斷走哪軌（Decision Tree）

```
這個任務需要改到 layout、字體家族、tokens、或新增路由嗎？
      │
      ├── 是 ──▶ 結構軌
      │
      └── 否 ──▶ 這個任務的輸出會被開發部署、或上到產品站嗎？
                        │
                        ├── 否（只是一張圖 / 一份素材） ──▶ 模板軌
                        │
                        └── 是 ──▶ 能用既有 templates/ 裡的模板 + tokens 拼出來嗎？
                                          │
                                          ├── 能 ──▶ 模板軌
                                          │
                                          └── 不能 ──▶ 結構軌
```

### 速查表

| 任務類型                             | 軌道                       |
| ------------------------------------ | -------------------------- |
| IG 活動圖                            | 模板軌                     |
| 七日書 landing（新活動但版型可沿用） | 模板軌                     |
| 七日書 landing（新版型）             | 結構軌                     |
| 加一個「密碼可視化切換」按鈕         | 結構軌                     |
| 作品頁字級調整                       | 結構軌                     |
| 換 newsletter header 圖              | 模板軌                     |
| Archive 視覺改版                     | 結構軌                     |
| 新增宣傳圖模板本身                   | 結構軌（因為是在建立模板） |

---

## 執行備忘

- **任何軌都必須使用 tokens**。若需要新 token，流程自動升級為結構軌。
- **handoff spec** 只在結構軌需要；模板軌以模板本身的變數定義取代。
- Agent（Claude / Codex）可在兩條軌都介入：模板軌用於填變數與產圖，結構軌用於設計生成、critique、handoff spec 產出。
